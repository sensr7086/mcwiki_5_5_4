# Quality Criteria — 100점 채점 기준

sub-skill `live` 승급의 1차 관문. **80점 미만 = re-draft.**

## 1. 4개 차원 (총 100점)

| 차원       | 배점 | 핵심 질문                                                    |
| -------- | -- | -------------------------------------------------------- |
| 정확성      | 30 | 영상/코드와 다른 진술이 없는가?                                       |
| 출처 추적성   | 20 | 모든 핵심 주장이 영상 타임스탬프/코드 라인/논문 페이지로 역추적 가능한가?               |
| 완전성      | 25 | §4.1 의 8개 절을 모두 다루고, 함정/관련 sub-skill cross-link 이 충실한가?  |
| 가독성/단순성  | 25 | Karpathy 어조 (담백, 직접) 를 따르며 사족이 없는가? 코드 5줄당 1줄 주석 정도인가?    |

## 2. 차원별 등급

### 2.1. 정확성 (30점)

| 등급        | 점수    | 기준                                                |
| --------- | ----- | ------------------------------------------------- |
| Excellent | 27–30 | 코드 실행 검증 완료. 차원·시드·하이퍼파라미터 영상과 일치.                |
| Good      | 21–26 | 개념적 정확. 하이퍼파라미터 일부 단순화 (이유 명시).                    |
| Marginal  | 15–20 | 작은 오류 1~2 (예: dim 표기 불일치, off-by-one). 본문에서 수정 가능. |
| Reject    | 0–14  | 핵심 개념 오해 (예: causal mask 방향 반대). 재초안.              |

### 2.2. 출처 추적성 (20점)

| 등급        | 점수    | 기준                                       |
| --------- | ----- | ---------------------------------------- |
| Excellent | 18–20 | 모든 핵심 주장에 S/B 등급 인용. 코드는 GitHub 라인까지.    |
| Good      | 14–17 | 주요 주장 인용 충분. 일부 보조 주장만 인용 누락.            |
| Marginal  | 10–13 | 영상 인용은 있으나 타임스탬프 누락. 또는 일반 주장 무근거.       |
| Reject    | 0–9   | 인용 거의 없음. C 등급 (블로그) 의존.                  |

### 2.3. 완전성 (25점)

8개 의무 절 (§4.1) 중 누락 1개당 -3점. cross-link 누락 -2점/회.

| 등급        | 점수    | 기준                                       |
| --------- | ----- | ---------------------------------------- |
| Excellent | 22–25 | 8개 절 + 함정 3개 이상 + 관련 sub-skill 3+ cross-link. |
| Good      | 17–21 | 8개 절 충족. 함정/cross-link 다소 빈약.            |
| Marginal  | 12–16 | 1~2 개 절 누락 또는 cross-link 거의 없음.          |
| Reject    | 0–11  | 절 다수 누락. 핸드오프 불가.                         |

### 2.4. 가독성/단순성 (25점)

| 등급        | 점수    | 기준                                                |
| --------- | ----- | ------------------------------------------------- |
| Excellent | 22–25 | 한 단락 한 아이디어. 코드 5줄당 1줄 주석. 한국어 본문, 영어 식별자 일관.    |
| Good      | 17–21 | 약간의 잉여 어구. 가독성 정상.                                |
| Marginal  | 12–16 | 사족 (예: "참고로", "한편") 빈발. 단락 길이 불균형.                |
| Reject    | 0–11  | 마케팅 톤, 과장, 모호한 대명사. 재작성.                          |

## 3. 채점 절차

1. 작성 직후 **자가 점검** (작성자 본인). 80 미만이면 발행 금지.
2. **별도 세션** 의 Evaluator 가 8단계 (`03_EvaluatorRecipe.md`) 로 재채점.
3. 두 점수의 **낮은 쪽** 을 기록 (보수적).
4. 점수 + 차원별 코멘트는 frontmatter `quality_score` 와 `quality_notes` 에 기록 (선택, 없어도 통과 가능).

## 4. 점수와 라이프사이클

| 점수 합산 | 처리                            |
| ----- | ----------------------------- |
| 90–100 | `status: live` 즉시 승급          |
| 80–89  | `status: live` 승급 + 개선 메모 남김  |
| 65–79  | `status: evaluated`. 한 번 더 보완. |
| 0–64  | `status: draft`. 재초안.         |

## 5. 자주 보이는 감점 사유

- 차원 표기를 `(B,T,C)` 통일 안 함 (Karpathy 표기와 다름) → 정확성 -3.
- "어텐션은 토큰 간 정보 교환" 같은 일반론에 인용 누락 → 출처 추적성 -3.
- 함정 절이 비어있음 → 완전성 -3.
- "사실은", "참고로" 남발 → 가독성 -2 ~ -5.
